🔍 Mini Log Analyzer com MCP

Roadmap de aprendizado — do protocolo ao loop completo

🧠01
Entender o MCP Conceito

Construir o modelo mental do protocolo antes de qualquer código. MCP é RPC sobre stdio — Claude inicia o server.py como processo filho e troca mensagens JSON pelo stdin/stdout.

list_tools() call_tool() stdio transport SDK mcp
✓ Sinal: conseguir explicar com suas palavras o que acontece quando Claude chama uma tool.
🗄️02
Gerar os dados falsos Dados

Criar fake_logs.json com 500–1000 registros realistas de logs Kubernetes, incluindo padrões como OOM errors, timeouts e connection refused.

fake_logs.json faker id, timestamp, level service, pod, namespace
✓ Sinal: arquivo gerado com distribuição razoável de níveis e namespaces.
⚙️03
Construir o server.py Core

Implementar 4 tools no MCP server com os padrões obrigatórios: summary_only, fields filter, bloco _meta e cap de resposta em 10KB.

search_logs count_logs get_log_by_id list_services summary_only _meta response cap
✓ Sinal: rodar server.py sem erro e testar manualmente via stdin.
🔌04
Configurar o .mcp.json Config

Registrar o server no Claude Code com caminhos absolutos para o Python e para o server.py.

.mcp.json caminhos absolutos type: stdio
✓ Sinal: rodar claude e ver o MCP server listado como conectado.
📘05
Escrever o SKILL.md Skill

Ensinar o Claude a usar as tools de forma custo-consciente: contar antes de buscar, usar fields filter, interpretar _meta e só fazer full fetch quando necessário.

SKILL.md plano de query interpretação _meta exemplos de uso
✓ Sinal: Claude responde "quais ferramentas você tem?" usando o conteúdo do SKILL.md.
🧪06
Testar o loop completo Teste

Fazer perguntas em linguagem natural e observar se Claude segue o plano do SKILL.md: usa summary_only, lê _meta, adapta as próximas chamadas.

perguntas em PT observar tool calls validar _meta loop
✓ Sinal: Claude responde sem fazer fetches desnecessários de documentos.
